## 1. 디자인 원칙 & 방향성

### 1.1 비주얼 톤 & 매너

**포지셔닝: "기술 전문가의 인간적 목소리"**

개발자/창업자 타겟의 개인 브랜딩 사이트는 두 가지 긴장 관계를 조율해야 한다.

- **전문성(Expertise)** vs **접근성(Approachability)**: 기술 능력을 증명하면서도 차갑지 않은 온기 유지
- **자기 표현(Personal Brand)** vs **제품 홍보(Product Promotion)**: 인간 서사가 주(主), 제품이 종(從)

CodeForge 홈페이지가 "제품 중심, 변환율 최적화" 톤이라면, 개인 브랜딩 사이트는 "사람 중심, 신뢰 구축" 톤으로 의도적으로 차별화한다.

**비주얼 참조 벤치마크:**

| 참조 대상 | 차용 요소 |
|-----------|-----------|
| Linear.app | 정밀한 타이포그래피, 미니멀 레이아웃, 다크 기반 색감 |
| Stripe.com/about | 인물 사진 + 텍스트 균형, 신뢰 신호 배치 |
| leerob.io | 1인 개발자 개인 브랜딩, 간결한 IA, 텍스트 중심 |
| rauno.me | 섬세한 마이크로인터랙션, 개성 있는 타이포그래피 |

### 1.2 디자인 시스템 성격 정의

**키워드:** Precise / Minimal / Warm / Confident

- **Precise**: 8px 그리드 기반 정렬, 픽셀 단위 일관성
- **Minimal**: 불필요한 장식 요소 제거, 콘텐츠가 인터페이스
- **Warm**: 순수 흑백이 아닌 살짝 따뜻한 뉴트럴, 인물 사진의 적극 활용
- **Confident**: 과한 설명 없이 사실 기반 주장, 충분한 여백으로 자신감 표현

### 1.3 브랜드 성격 반영 방안

| 브랜드 성격 | 디자인 반영 방식 |
|-------------|----------------|
| Professional but Approachable | 깔끔한 Sans-serif 본문 + 개발자 프로필 사진 전면 배치 |
| Action-Oriented | "현재 진행형" 동사 CTA, 타이핑 애니메이션으로 생동감 |
| Transparent | 실패/도전 경험을 Timeline에 포함, 수치는 구체적으로 표기 |
| Data-Driven | 통계 수치를 Hero 하단 신뢰 지표 영역에 배치 |

### 1.4 CodeForge와의 의도적 차별화

| 항목 | CodeForge 홈페이지 | 개인 브랜딩 사이트 |
|------|-------------------|-------------------|
| 주인공 | 제품(AI 코드 생성) | 사람(개발자 창업자) |
| 핵심 감정 | 효율 / 흥분 | 신뢰 / 공감 |
| 주요 CTA | 무료 시작 / 전환 | 연락하기 / 더 알아보기 |
| 이미지 스타일 | 코드 데모, UI 스크린샷 | 인물 사진, 핸드라이팅 느낌 |
| 텍스트 밀도 | 중간 (기능 설명 필요) | 낮음 (여백 = 자신감) |

## 2. 타이포그래피 시스템

### 2.1 폰트 패밀리 선택

**EN (영문):** Geist — Vercel이 개발한 개발자 친화적 Sans-serif. 코드 가독성과 본문 가독성을 모두 만족하며 CodeForge와 동일한 폰트 패밀리로 시각적 일관성 유지.

**KO (한국어):** Pretendard — 한국어 웹 타이포그래피의 사실상 표준. 글자 간격, 세로 정렬, 굵기 계층이 Geist와 잘 조화됨.

**Mono (코드):** Geist Mono — 코드 스니펫, 기술 스택 태그, 터미널 스타일 표현에 사용.

```css
/* font-family 정의 */
--font-sans: 'Geist', 'Pretendard', -apple-system, sans-serif;
--font-mono: 'Geist Mono', 'JetBrains Mono', monospace;
```

### 2.3 EN/KO 이중 언어 대응 기준

| 항목 | 영문(EN) | 한국어(KO) |
|------|---------|-----------|
| **줄 높이 조정** | 기본값 사용 | +0.1 추가 (한글 자소 높이 보정) |
| **자간 조정** | 기본값 | 0 (한글은 자간 조절 불필요) |
| **폰트 크기 조정** | 기본값 | 동일 (Pretendard의 높은 가독성) |
| **단어 줄바꿈** | `word-break: break-word` | `word-break: keep-all` (의미 단위 유지) |
| **최대 줄 길이** | 65-75 characters | 25-35 글자 |

```css
/* 한국어 본문 전용 규칙 */
:lang(ko) p, :lang(ko) li {
  word-break: keep-all;
  line-height: 1.8; /* EN 대비 +0.1~0.15 */
}
```

## 3. 컬러 시스템

### 3.1 기존 디자인 토큰 (CodeForge와 공유)

| 토큰명 | Light | Dark | 용도 |
|--------|-------|------|------|
| `--color-background` | `#ffffff` | `#0a0a0a` | 페이지 배경 |
| `--color-foreground` | `#171717` | `#ededed` | 기본 텍스트 |
| `--color-primary` | `#2563eb` | `#3b82f6` | 링크, 강조 포인트 |
| `--color-muted-foreground` | `#71717a` | `#a1a1aa` | 부가 텍스트, 플레이스홀더 |
| `--color-border` | `#e5e5e5` | `#262626` | 구분선, 카드 테두리 |

### 3.2 개인 브랜딩 사이트 전용 추가 토큰

**추가 시멘틱 컬러 — 필요 최소한으로 확장:**

| 토큰명 | Light | Dark | 용도 |
|--------|-------|------|------|
| `--color-surface` | `#fafafa` | `#111111` | 카드, 섹션 배경 (background보다 살짝 다름) |
| `--color-surface-hover` | `#f4f4f5` | `#1a1a1a` | 카드 hover 상태 |
| `--color-accent` | `#f59e0b` | `#fbbf24` | Timeline 점, 현재 포커스 표시 |
| `--color-success` | `#10b981` | `#34d399` | 완료된 프로젝트 배지, 폼 성공 상태 |
| `--color-subtle` | `#f0f0f0` | `#1e1e1e` | Skills 태그 배경, 코드 블록 배경 |
| `--color-subtle-foreground` | `#404040` | `#c0c0c0` | Skills 태그 텍스트 |

**추가 배경 컬러 (섹션 분리용):**

| 토큰명 | Light | Dark | 용도 |
|--------|-------|------|------|
| `--color-section-alt` | `#f9f9f9` | `#0f0f0f` | 짝수 번째 섹션 배경 (background와 미묘하게 대비) |

### 3.3 컬러 사용 원칙

1. **primary 컬러(파랑)는 절제**: 링크, 포커스 링, 중요 강조에만 사용. 버튼은 foreground(검정/흰색) 사용.
2. **accent 컬러(앰버)는 포인트**: Timeline 현재 위치, 중요 수치 하이라이트에만.
3. **배경 변화는 subtle하게**: Light 모드에서 `#ffffff` → `#fafafa` 수준의 대비로 섹션 구분.
4. **다크 모드 우선 설계**: 개발자 타겟은 다크 모드 사용률이 높음. 다크 모드에서 먼저 검토 후 라이트 조정.

### 3.4 Light/Dark 토큰 전체 명세

```css

  --color-foreground: #171717;

  --color-border: #e5e5e5;

  /* 개인 브랜딩 추가 토큰 */

  --color-surface-hover: #f4f4f5;
  --color-accent: #f59e0b;
  --color-success: #10b981;

.dark {

  --color-muted-foreground: #a1a1aa;
  --color-border: #262626;

  /* 개인 브랜딩 추가 토큰 */
  --color-surface: #111111;
  --color-surface-hover: #1a1a1a;
  --color-accent: #fbbf24;
```

## 4. 컴포넌트 명세

### 4.1 Navigation (GNB)

#### 구조

```
┌─────────────────────────────────────────────────────────┐
│  [로고/이니셜]    Home  About  Products  Projects  Contact  [EN|KO]  [다크모드]  │
└─────────────────────────────────────────────────────────┘
```

**구성 요소:**
- 로고 영역: 이니셜 + 이름 텍스트 (예: "JD / John Doe")
- 중앙 네비게이션 링크 5개
- 우측: 언어 토글(EN/KO) + 다크모드 토글

**레이아웃 규칙:**
- `position: sticky; top: 0` — 스크롤 시 상단 고정
- 배경: `background/80` + `backdrop-blur-md` — 스크롤 시 반투명 처리
- 높이: 64px (데스크톱), 56px (모바일)
- 최대 너비: 1280px, 좌우 패딩: 24px (데스크톱), 16px (모바일)
- 스크롤 진입 시 `border-bottom: 1px solid border` 추가

#### 상태 명세

| 상태 | 스타일 |
|------|--------|
| **Default** | foreground 텍스트, weight 400 |
| **Active (현재 페이지)** | foreground 텍스트, weight 600, 하단 2px border accent 컬러 |
| **Hover** | opacity 0.7, transition 150ms ease |
| **Focus** | outline: 2px solid primary, outline-offset: 4px |
| **Scrolled** | backdrop-blur + border-bottom 추가 |

#### 모바일 대응 (768px 미만)

```
┌─────────────────────────────┐
│  [로고/이니셜]          [☰]  │
└─────────────────────────────┘

[메뉴 열림 시]
┌─────────────────────────────┐
│  [로고/이니셜]          [X]  │
├─────────────────────────────┤
│  Home                       │
│  About                      │
│  Products                   │
│  Projects                   │
│  Contact                    │
├─────────────────────────────┤
│  [EN]  [KO]    [다크모드]   │
└─────────────────────────────┘
```

- 햄버거 메뉴 아이콘: Lucide `Menu` / 닫기: Lucide `X`
- 열림 애니메이션: 상단에서 하단으로 슬라이드 (200ms ease-out)
- 메뉴 항목 높이: 최소 56px (터치 타겟 기준)
- 배경: background 100% (반투명 없이 완전 불투명)
- 메뉴 외부 클릭 시 닫힘

#### Variant

| Variant | 설명 | 사용 위치 |
|---------|------|-----------|
| `default` | 기본 투명 배경 | Hero 위 초기 상태 |
| `scrolled` | 반투명 blur 배경 + 테두리 | 스크롤 후 |
| `mobile-open` | 전체 화면 드로워 | 모바일 메뉴 열림 |

## 6. 인터랙션 & 애니메이션 명세

### 6.1 전체 원칙

| 항목 | 기준값 |
|------|--------|
| **기본 Duration** | 150ms (마이크로), 200ms (요소), 300ms (페이지) |
| **기본 Easing** | ease-out (등장), ease-in (퇴장), ease-in-out (전환) |
| **최대 Duration** | 500ms (복잡한 레이아웃 전환) |
| **모션 원칙** | 목적 있는 움직임만. 장식적 애니메이션 지양 |

### 6.2 페이지 전환 애니메이션

Next.js App Router 기반 페이지 전환:

| 속성 | 값 |
|------|----|
| **방식** | Fade + subtle upward slide |
| **Duration** | 300ms |
| **Easing** | ease-out |
| **구현** | `opacity: 0→1`, `translateY: 8px→0` |

```css
/* 페이지 등장 */
@keyframes page-enter {
  from { opacity: 0; transform: translateY(8px); }
  to   { opacity: 1; transform: translateY(0); }
}

.page-wrapper {
  animation: page-enter 300ms ease-out;
}
```

### 6.3 Scroll 기반 등장 애니메이션

Intersection Observer API 사용. `view-timeline` CSS 또는 Framer Motion 미사용 시 직접 구현.

| 요소 | 트리거 | Duration | Easing | 효과 |

- 링크 클릭 시 닫기

- 드로워: `role="dialog"`, `aria-modal="true"`
- 포커스 트랩: 메뉴 열림 시 탭 포커스를 드로워 내부로 제한
- 첫 포커스: 닫기(X) 버튼으로 이동

**커스텀 Tailwind 애니메이션 (tailwind.config.js):**
```javascript
extend: {
  keyframes: {
    'fade-in': {
      from: { opacity: '0', transform: 'translateY(8px)' },
      to: { opacity: '1', transform: 'translateY(0)' },
    },
    'slide-up': {
      from: { opacity: '0', transform: 'translateY(16px)' },
      to: { opacity: '1', transform: 'translateY(0)' },
    },
    'blink': {
      '0%, 100%': { opacity: '1' },
      '50%': { opacity: '0' },
    },
  },
  animation: {
    'fade-in': 'fade-in 300ms ease-out',
    'slide-up': 'slide-up 400ms ease-out',
    'blink': 'blink 1s step-end infinite',
  },
}
```

## 10. CodeForge 홈페이지와의 연계 전략

### 10.1 두 사이트의 관계 정의

```
개인 브랜딩 사이트 (본 사이트)
    ↓ "제품 보기" CTA
CodeForge 홈페이지 (제품 사이트)
    ↓ "만든 사람" 링크
개인 브랜딩 사이트 About 페이지
```

**전략:** 개인 브랜딩 사이트는 CodeForge의 "신뢰 백업" 역할. CodeForge 방문자가 "이 사람은 누구?" 라는 질문을 가질 때 자연스럽게 연결되어야 함.

### 10.2 공유 디자인 토큰

두 사이트가 공유하는 토큰은 단일 소스로 관리한다.

| 공유 항목 | 관리 위치 | 동기화 방법 |
|-----------|-----------|------------|
| 컬러 토큰 | `globals.css` (각 사이트) | 복사 유지 또는 공유 npm 패키지 |
| Border radius | `tailwind.config.js` | 동일 설정 |
| Font family | `layout.tsx` | 동일 폰트 로드 방식 |
| 버튼 Variant 규칙 | 각 사이트 Button 컴포넌트 | 스타일 동일, 코드 별도 관리 |

**장기 전략 (Phase 2):** 공유 컴포넌트를 내부 디자인 시스템 패키지(`@brand/ui`)로 추출하여 두 사이트에서 import.

### 10.3 공유 컴포넌트 재사용 가이드

| 컴포넌트 | 재사용 가능 여부 | 주의사항 |
|----------|----------------|---------|
| `Button` | 재사용 가능 | Primary variant 스타일 동일 |
| `Navigation` | 부분 재사용 | 링크 구성이 다름 — 로직 분리 필요 |
| `Footer` | 부분 재사용 | 링크와 설명 텍스트 다름 |
| `ThemeToggle` | 재사용 가능 | 동일 구현 |
| `LanguageToggle` | 재사용 가능 | 동일 구현 |
| `Badge` | 재사용 가능 | Pill 스타일 |
| `SkipLink` | 재사용 가능 | i18n 적용 필수 |
| `ProjectCard` | 개인 브랜딩 전용 | CodeForge에는 없음 |
| `Timeline` | 개인 브랜딩 전용 | CodeForge에는 없음 |
| `ContactForm` | 별도 관리 | 폼 필드 구성이 다름 |

### 10.4 상호 링크 전략

**개인 브랜딩 → CodeForge:**
- Navigation: "Products" 링크 → CodeForge 홈
- Home Featured Section CTA → CodeForge 홈
- Products 페이지 전체 = CodeForge 소개

**CodeForge → 개인 브랜딩:**
- Footer에 "Made by [이름]" 링크
- About 페이지(CodeForge)에 창업자 링크

### 10.5 SEO 연계

| 항목 | 개인 브랜딩 | CodeForge |
|------|-----------|-----------|

## 용어집 (Glossary)

| 용어 | 정의 |
|------|------|
| **GNB** | Global Navigation Bar. 사이트 전체에 공통으로 표시되는 상단 네비게이션 |
| **CTA** | Call to Action. 사용자에게 특정 행동을 유도하는 버튼/링크 |
| **Breakpoint** | 반응형 레이아웃이 변화하는 화면 너비 기준점 |
| **Easing** | 애니메이션의 가속/감속 곡선. ease-out은 빠르게 시작해 천천히 끝남 |
| **Stagger** | 여러 요소가 순차적으로 딜레이를 두고 등장하는 애니메이션 기법 |
| **Trust Signal** | 사용자 신뢰를 높이는 요소. 사용자 수치, 후기, 언론 보도 등 |
| **Semantic HTML** | 의미를 가진 HTML 태그 사용. `<nav>`, `<main>` 등 |
| **ARIA** | Accessible Rich Internet Applications. 스크린 리더를 위한 접근성 속성 |
| **Surface** | 카드, 패널 등 배경보다 약간 다른 컬러의 UI 레이어 |
| **Foreground** | 텍스트와 같이 배경 위에 올라오는 UI 요소의 컬러 |
| **Token** | 디자인 시스템의 최소 단위. 컬러, 간격, 폰트 크기 등의 변수 |
| **IA** | Information Architecture. 정보 구조. 사이트의 콘텐츠 계층 구조 |
| **A11y** | Accessibility의 약어 (a + 11글자 + y) |
| **safe-area-inset** | iOS 기기의 노치/홈 인디케이터를 피하기 위한 여백 환경 변수 |
