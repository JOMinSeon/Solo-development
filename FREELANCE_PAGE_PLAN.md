# 외주 의뢰 페이지 기획서 (/products → /freelance)

**작성일**: 2026-03-27
**프로젝트**: 개발자 포트폴리오 홈페이지
**대상 페이지**: `/products` → 외주(프리랜서) 의뢰 페이지로 전환
**기술 스택**: Next.js, TypeScript, Tailwind CSS, Lucide React

---

## 1. 페이지 목적 및 핵심 메시지

### 페이지 목적
외주 프로젝트를 직접 의뢰하려는 고객과 협업파트너를 위한 **신뢰도 높은 의뢰 유도 페이지**

### 핵심 메시지 (Value Proposition)
> **"웹 개발의 모든 단계를 혼자 마스터한 풀스택 개발자와 함께라면, 복잡한 프로젝트도 명확하게 완성됩니다."**

### 서브 메시지
- 기획부터 런칭까지 통합 관리 (스타트업/SME 경험)
- 사용자 경험을 최우선으로 하는 개발 (UX-first)
- 빠른 커뮤니케이션과 투명한 진행 보고 (주간 리뷰, 마일스톤)
- 기술 부채 없는 깔끔한 코드 (유지보수 용이)

**왜 이 메시지인가?**
- 직접의뢰 고객은 "싼 개발자"가 아니라 "신뢰할 수 있는 개발자"를 찾음
- 과장된 마케팅보다 **실무 경험과 구체성**이 의사결정에 영향
- 채용담당자도 존재하므로 자신감 있는 톤(vs 의존적 톤) 유지

## 2. 페이지 섹션 구조 및 흐름

```
┌─────────────────────────────────────────────────────────┐
│ 1. Hero Section                                         │
│   - 페이지 타이틀, 핵심 메시지, 서브 메시지            │
│   - CTA 버튼 (상담 예약, 포트폴리오 보기)             │
├─────────────────────────────────────────────────────────┤
│ 2. Why Me? (차별화 포인트)                             │
│   - 3개 핵심 강점 카드                                 │
│   - 통계/증명 (완료 프로젝트, 만족도, 기업)           │
├─────────────────────────────────────────────────────────┤
│ 3. Service Offerings (외주 서비스 종류)                │
│   - 4개 서비스 카테고리 (Web, App, 통합, 컨설팅)     │
│   - 각 서비스별 설명, 예상 기간, 포함 사항            │
├─────────────────────────────────────────────────────────┤
│ 4. My Process (프로젝트 진행 프로세스)                 │
│   - 5단계 플로우 (상담→기획→개발→테스트→런칭)       │
│   - 각 단계별 산출물, 예상 기간                        │
├─────────────────────────────────────────────────────────┤
│ 5. Pricing (요금 체계)                                 │
│   - 프로젝트형 vs 시간제 설명 및 비교                 │
│   - 가격대 예시 (소규모/중규모/대규모)                │
│   - 추가 서비스 (유지보수, 컨설팅)                    │
├─────────────────────────────────────────────────────────┤
│ 6. Success Stories (사례/후기)                         │
│   - 완료한 프로젝트 3~4개 사례 (간단 소개)            │
│   - 고객사/팀, 결과, 기술 스택, 링크                  │
├─────────────────────────────────────────────────────────┤
│ 7. FAQ (자주 묻는 질문)                                 │
│   - 일정, 비용, 커뮤니케이션, 기술, 보안 관련        │
├─────────────────────────────────────────────────────────┤
│ 8. Final CTA (최종 액션)                               │
│   - 상담 예약, 이메일 문의, 카톡/전화                 │
│   - 응답 시간 명시 (24시간 이내)                      │
└─────────────────────────────────────────────────────────┘
```

### 3.1 Hero Section
**목표**: 30초 내 페이지의 목적과 가치를 명확히 전달

**레이아웃**:
- 텍스트 중심 (좌측) + 이미지/시각 요소 (우측)
또는 전체 너비 센터 레이아웃

**콘텐츠**:
```typescript

  badge: "현재 외주 가능" | "상담 중심 활동";

  description: string; // 2~3줄, 핵심 가치
  images?: {
    desktop: string; // 데스크탑용
    mobile: string;  // 모바일용
  };
  cta: {
    primary: { text: "상담 예약", href: "#contact" };

  };
}
```

## 6. 차별화 포인트 (경쟁사 대비)

### 6.1 "기획부터 런칭까지 단일 책임"
- **차별점**: 대부분의 외주 개발사는 "개발만" 담당. 조민성은 기획, 디자인, 개발, 배포까지 통합 관리
- **증명**: PetCare, B2B SaaS, 헬스케어 등 4개 자체 서비스 런칭 경험
- **메시지**: "스타트업처럼 생각하고, 싱글 제목 개발자처럼 책임진다"

### 6.2 "UX-First 개발"
- **차별점**: 아름다운 UI + 빠른 성능 + 직관적 흐름
- **증명**: Tailwind CSS 마스터, 반응형 디자인 필수, 웹 바이탈 최적화
- **메시지**: "사용자가 사랑하는 제품을 만든다"

### 6.3 "장기 유지보수 관점의 개발"
- **차별점**: "빠르게 만드는" 개발이 아니라 "오래 쓸 수 있는" 개발
- **증명**: TypeScript, 명확한 코드 구조, 개발 문서 제공
- **메시지**: "런칭 후 3년을 생각하고 코드를 짠다"

### 6.4 "투명한 커뮤니케이션"
- **차별점**: 주간 리포트, GitHub 접근 권한, 진행 회의

- **장점**: 즉시 소통, 신뢰도 상승

## 9. 작성 톤(Tone) & 스타일 가이드

### 9.1 톤
- **신뢰도**: 과장 금지, 실제 경험 기반
- **친근함**: 형식적이지 않되, 전문성 유지
- **명확성**: 모호한 표현 금지, 구체적 수치 제시
- **자신감**: 강한 주장 (약한 표현 최소화)

### 9.2 피해야 할 표현
- ❌ "최고의", "최강의", "가장 저렴한" (증명 불가)
- ❌ "여러 팀이 함께 작업합니다" (1인이라는 것이 차별점)
- ❌ "24시간 개발" (현실적이지 않음)
- ❌ 과도한 이모지 또는 시각 효과

### 9.3 권장 표현
- ✅ "1인 풀스택 개발자의 장점"
- ✅ "4개 자체 서비스 런칭 경험"
- ✅ "프로젝트별 맞춤 솔루션"
- ✅ "명확한 일정과 가격 제시"
