# 1인 개발자 소개 페이지 기획안

## 1. 페이지 목적 및 목표

**핵심 목적:**
개발자의 기술력, 프로젝트 경험, 전문성을 명확하고 신뢰감 있게 전달하여 채용담당자, 협업 파트너, 고객이 "이 개발자와 함께 일하고 싶다"는 확신을 갖도록 하는 것

**구체적 목표:**
- **인지**: 방문자가 30초 이내에 "이 사람이 누구이고 무엇을 하는 사람인지" 파악
- **신뢰**: 과거 프로젝트와 기술 스택을 통해 능력 입증
- **행동 유도**: 이메일 문의, LinkedIn/GitHub 연결, 프로젝트 클릭 등의 CTA(Call-To-Action) 완수
- **SEO**: 검색 엔진에 최적화되어 관련 키워드로 유입 증대

---

## 2. 타겟 유저

| 세그먼트 | 비율 | 방문 목적 | 핵심 니즈 |
|--------|-----|---------|---------|
| **채용담당자/헤드헌터** | 40% | 채용 후보 평가 | 기술 수준, 경력, 협업 스타일 빠르게 파악 |
| **협업 파트너(프리랜서/에이전시)** | 30% | 협업 가능성 평가 | 관련 경험, 커뮤니케이션 방식, 포트폴리오 |
| **고객(직접 의뢰)** | 20% | 용역 여부 판단 | 사례, 비용, 기간, 신뢰성 |
| **동료 개발자** | 10% | 네트워킹, 기술 벤치마킹 | 프로젝트 스택, 성장 여정, 블로그/인사이트 |

## 3. 페이지 구조 (섹션별)

### 3.1 Hero Section (랜딩)
**목적**: 3초 이내에 개발자의 정체성과 가치 제안을 명확히 전달

**구성 요소:**
- **메인 헤드라인**: "안녕하세요. [이름]입니다 / Full-Stack Developer" (또는 직무)
  - 부제: 1줄의 핵심 포지셔닝 (예: "복잡한 문제를 우아한 코드로 해결하는 개발자")
- **배경**: 미니멀한 그라디언트 또는 파티클 애니메이션
- **프로필 이미지**: 100x100px, 원형, 신뢰감 있는 얼굴 사진
- **CTA 버튼**: "프로젝트 보기" / "이력서 다운로드" / "연락하기" (주요 CTA 1-2개)
- **스크롤 인디케이터**: 아래로 더 있다는 신호

**내용 예시:**
```
안녕하세요, 홍길동입니다.

Full-Stack Developer | 사용자 경험을 최우선으로 하는 개발자

복잡한 문제를 간단한 코드로 풀어내고,
그 코드가 사람들의 삶을 조금이라도 나아지게 만드는 것을 좋아합니다.

[프로젝트 보기]  [연락하기]
```

---

### 3.2 About Section (자기소개)
**목적**: 개발자의 경험, 철학, 성장 여정을 스토리텔링으로 전달

**구성 요소:**
- **한 문단 자기소개** (150-200자):
  - "XX년 개발 경험, Y개 프로젝트 런칭, Z개 기술 스택"
  - 개발자의 핵심 가치관 1-2개 (예: "사용자 피드백 중심", "코드 품질 집착")
- **주요 성과 3-4개** (아이콘 + 수치):
  - "3년+ 개발 경험"
  - "15+ 프로젝트 완료"
  - "50+ 고객 만족도 100점"
  - "1M+ 코드라인 작성"
- **핵심 가치 3개** (아이콘 + 텍스트):
  - "빠른 학습": 새로운 기술을 빠르게 습득하고 적용
  - "사용자 중심": 항상 사용자의 입장에서 생각
  - "책임감": 마감일과 품질을 동시에 지키기

---

### 3.3 Skills Section (기술 스택)
**목적**: 보유한 기술을 카테고리별로 명확히 제시하여 신뢰도 향상

**구성 요소:**
- **카테고리별 기술 분류**:
  - Frontend: React, Next.js, TypeScript, Tailwind CSS
  - Backend: Node.js, Express, Python, FastAPI
  - Database: PostgreSQL, MongoDB, Redis
  - Infra & DevOps: Docker, GitHub Actions, AWS
  - Tools: Git, Figma, Jira, VS Code
- **숙련도 표시**: Expert / Advanced / Intermediate
- **시각화**: 태그, 배지, 또는 카드 형태

---

### 3.4 Projects Section (포트폴리오)
**목적**: 실제 프로젝트를 통해 기술력과 문제 해결 능력 입증

**구성 요소:**
- **프로젝트 카드** (각 카드는):
  - 프로젝트 썸네일 이미지 (또는 스크린샷)
  - 프로젝트명
  - 한 줄 요약 (30자 이내): "어떤 문제를 해결했는가"
  - 기술 스택 태그 (3-5개)
  - 날짜 (YYYY.MM)
  - "보기" 링크 (프로젝트 상세 페이지 또는 GitHub/Live Demo)
- **필터링 기능**: "All" / "Frontend" / "Backend" / "Full-Stack" 탭
- **정렬**: 최신 프로젝트 상단에 배치

**프로젝트 데이터 구조:**
```json
{
  "id": "project-1",
  "title": "이커머스 플랫폼",
  "description": "다중 판매자를 지원하는 실시간 주문 시스템",
  "thumbnail": "/images/project-1.png",
  "tags": ["React", "Node.js", "PostgreSQL", "Stripe"],
  "date": "2024.11",
  "link": "/projects/ecommerce",
  "github": "https://github.com/...",
  "live": "https://example.com"
}
```

### 3.5 Contact / CTA Section (연락처)
**목적**: 방문자가 쉽게 연락할 수 있는 진입점 제공

**구성 요소:**
- **헤드라인**: "함께 프로젝트를 진행해보세요"
- **연락 수단** (2-3개):
  - 이메일 (클릭하면 메일 클라이언트 열림)
  - LinkedIn 링크
  - GitHub 링크
  - (Optional) Calendly 링크 (일정 예약)
- **간단한 문의 폼**: 이름, 이메일, 메시지, 문의 타입 선택

## 5. UX/UI 방향성

### 5.1 디자인 원칙
- **미니멀리즘**: 불필요한 요소 제거, 공백 활용으로 가독성 극대화
- **일관성**: 컬러, 타이포그래피, 간격이 전체 페이지에서 일관됨
- **접근성**: 명암비 충분, 터치 영역 최소 44px, 키보드 네비게이션 지원
- **성능**: 로딩 속도 < 3초, 누적 레이아웃 시프트 < 0.1

### 5.2 컬러 팔레트 (추천: 모던/테크)
- 메인: `#0F172A` (진한 네이비)
- 강조: `#06B6D4` (사이언) 또는 `#3B82F6` (블루)
- 배경: `#FFFFFF` (라이트) / `#111827` (다크)
- 텍스트: `#1F2937` (라이트) / `#F3F4F6` (다크)

### 5.3 타이포그래피
- **영문**: Inter 또는 Geist Sans
- **한글**: Pretendard 또는 SUIT
- **크기 스케일**: H1 48-56px / H2 32-40px / H3 24-28px / Body 14-16px
- **라인 높이**: 1.5-1.8

### 5.4 분위기
- **감정**: 신뢰, 전문성, 친근함
- **톤**: 너무 거창하지 않되, 역량을 명확히 드러냄
- **레이아웃**: 깔끔, 계층적, 시각적 호흡감

1. 문의 폼 UI (React Hook Form)
